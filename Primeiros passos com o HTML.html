Ferramentas utilizadas

(Voltar)

Tags"h" serve para títulos, variando entre seis níveis hierarquicos que são:

h1

h2

h3

h4

h5

h6

É importante dividir os títulos deixando o h1 apenas para o título principal, h2 ou h3 para títulos das seções e h4 a h6 para subtítulos ou títulos que possuam baixo valor hierárquico em relação aos demais

Tag "mark" serve para representar um trecho de destaque em um texto como por exemplo o texto acima.

Tag "small" Define um texto menor.

Tag "i" é frequentemente usada para indicar um termo técnico, uma frase de outro idioma, um pensamento, um nome de navio, etc. O conteúdo interno é exibido em itálico

Tag "u" representa algum texto não articulado e com estilo diferente do texto normal, como palavras com erros ortográficos ou nomes próprios em texto chinês. O conteúdo interno geralmente é exibido com um sublinhado.

Tag "strong" é usada para definir o texto com forte importância. O conteúdo interno geralmente é exibido em negrito .

Tag "ol" define uma lista ordenada. Uma lista ordenada pode ser numérica ou alfabética.

Tag "ul" define uma lista não ordenada (marcada).

Tag "li" define um item de lista. É usada dentro de listas ordenadas, listas não ordenadas e em listas de menu.

Tag "a" define um hiperlink, que é usado para vincular de uma página a outra. O atributo mais importante deste elemento é o href; atributo que indica o destino do link.

Tag "hr" define uma quebra temática em uma página HTML (por exemplo, uma mudança de tópico). Geralmente é exibido como uma régua horizontal usada para separar o conteúdo (ou definir uma alteração) em uma página HTML.

Tag "sub" define o texto subscrito. O texto subscrito aparece meio caractere abaixo da linha normal e, às vezes, é renderizado em uma fonte menor. O texto subscrito pode ser usado para fórmulas químicas, como H2O.

Tag "sup" define o texto sobrescrito. O texto sobrescrito aparece meio caractere acima da linha normal e, às vezes, é renderizado em uma fonte menor. O texto sobrescrito pode ser usado para notas de rodapé, como WWW[1]

Tag "blockquote" é uma tag que especifica uma seção citada de outra fonte.

Tag"font" é utilizado para configurar o tipo, cor e tamanho das letras de um texto de uma página HTML OBS:É importante ressaltar que dentro do contexto do HTML 4.01 o marcador ""font" esta em desuso e será obsoletado no HTML 5.

Tag"del" representa uma parte do texto que foi excluída de um documento. Este elemento é (não necessariamente) renderizado pelos navegadores com uma linha entre o texto.

Tag"abbr" epresenta uma abreviação e opcionalmente fornece uma descrição completa para ela. Exemplo E.C.V é o melhor time da Bahia.

Inspetor de Elementos

(Voltar)

A ferramenta de inspecionar elemento é um utilitário que permite visualizar o código-fonte subjacente de qualquer página web. Além disso, você pode usá-la para fazer alterações temporárias e ver os resultados em tempo real, deixando o código-fonte original intacto

Isso é extremamente útil se você precisa testar uma alteração ou diagnosticar um problema. Também pode ser de grande valia se você encontrar um site com um recurso de que goste e estiver curioso para saber como ele foi implementado.

A maioria dos principais navegadores, incluindo Chrome, Firefox e Safari, oferece algumas variações dessa ferramenta

Quais são os recursos da ferramenta para inspecionar elemento?

  1. Localiza classes CSS em um site
  2. Soluciona problemas de um site
  3. Edita o texto para visualizar as alterações
  4. Testa um site usando emulação de dispositivo

Estrutura básica do HTML

(Voltar)

O documento HTML sempre inicia com o que chamamos de estrutura básica. Esta estrutura é quase que imutável. Sempre será dessa forma e você sempre, sempre começará seu HTML começando por esse código. Geralmente os editores como o Sublime Text já tem atalhos para iniciar os documentos HTMLs com essa estrutura, logo, você não precisa se preocupar em decorá-la, mas é bom que faça

É possível compreender o documento em HTML de uma maneira muito simples, através de uma divisão de blocos das tags essenciais, conforme a a seguinte estrutura:

  • Definição do documento (doctype)
  • Cabeça (head)
  • Corpo (body)

Veja abaixo como ela se inicia:

Doctype - Definindo o documento

O doctype não é uma tag HTML, mas uma instrução para o navegador e outros programas que podem ler seu site, que o código encontrado ali é um código HTML. Assim eles sabem o que fazer para mostrar seu site da melhor forma possível. Lembre-se: o doctype é OBRIGATÓRIO e deve ser sempre a PRIMEIRA LINHA do seu documento.

HEAD

Contém informações que não são transpostas visivelmente para o usuário/leitor do documento. São dados implícitos, de uso e controle do documento: vinculação com outros arquivos, aplicação de lógica de programação de scripts e metadados. Na prática, todo o conteúdo do cabeçalho fica delimitado entre a abertura e fechamento tag head.

BODY

Trata-se do documento em si, ou seja, a informação legível para o usuário/leitor do documento. É todo e qualquer texto que se deseja apresentar, assim como toda e qualquer forma de mídia de saída (imagens, sons, miniaplicativos embutidos, conteúdo multimídia, etc). Além disso, toda a apresentação de entrada de dados (formulários) também se aplica neste seção do documento. Na prática, o corpo do documento é delimitado pelo par de tags body/body.

Este é o preceito básico que deve estar muito bem claro para você: onde as marcações se aplicam, e quais são os resultados deste modelo. Por exemplo: se vocês deseja informar conteúdo textual para saída legível ao usuário do seu sistema web, esta marcação deverá obrigatoriamente estar no bloco do corpo da página. Ainda: para definir qual o tipo de codificação da página (uma meta informação do documento), esta deve obrigatoriamente estar marcada no cabeçalho do mesmo documento.

Dentro do elemento BODY sua estrutura de página terá os elementos semânticos da construção da sua página, onde serão declarados e identificados cabeçalhos, rodapé, conteúdo principal, etc.

Falando sobre TAGS

(Voltar)

As tags são usadas para informar ao navegador a estrutura do site. Ou seja: quando se escreve um código em HTML, as tags serão interpretadas pelo navegador, produzindo assim a estrutura e o conteúdo visual da página.

A principal característica das tags é estarem sempre dentro dos sinais de chevron (sinal de “maior que” e “menor que”), ou seja: < >.

As tags HTML são divididas em dois tipos: as que precisam de fechamento e as que não precisam de fechamento. As tags que precisam de fechamento possuem a sintaxe <"tag"> <"/tag">, já as que não precisam de fechamento possuem como estrutura .

Além disso, uma mesma tag pode receber um ou mais atributos, que possuirá um valor que modifica sua estrutura ou funcionalidade.

Atributos básicos

(Voltar)

Os atributos são usados para personalizar as tags, modificando sua estrutura ou funcionalidade. Igualmente, os atributos são utilizados para atribuir uma classe ou id a um elemento.

A maioria das tags tem seus próprios atributos. Contudo, existem alguns atributos genéricos que podem ser utilizados na maioria das tags HTML, vamos estudá-los:

  • class=”…“ – Atribui uma classe ao elemento (uma classe pode ser utilizada para um ou mais elementos);
  • id=”…“ – Atribui um id ao elemento (um id deve ser único, ou seja atribuído a um único elemento);
  • style=”…” – Permite incluir elementos CSS (estilos) dentro da tag;
  • lang=”…” – Define o idioma principal do elemento;
  • title=”…” – Define o título do elemento;
  • alt=”…” – Define um texto alternativo e, por isso, é muito utilizado em imagens, auxilia nas práticas de SEO;
  • hidden – Oculta o elemento;
  • align=”…” – Permite definir o padrão de alinhamento desse elemento, como por exemplo: right, center, left e justify;
  • width=”…” – Define uma largura para o elemento;
  • height=”…” – Define uma altura para o elemento.

Atributos HTML são palavras especiais usadas dentro da tag de abertura para controlar o comportamento do elemento. Os atributos HTML são um modificador de um tipo de elemento HTML. Com eles podemos identificar melhor um elemento, informar qual arquivo aquela tag deve utilizar, indicar o tipo de um campo de texto, etc…

Há dois tipos de atributos no HTML, os globais que são aceitos por todas as tags, como por exemplo: class, id, lang, style emtre outrs que você pode conferir no site https://www.w3schools.com . Também existem os específicos que somente algumas tags possuem, como src, disabled, href, label, etc…

São os principais atributos:

class=”NomeDaClasse”

Classes são como classificações de uma tag/elemento, para que no CSS estilizar uma tag específica, ou um conjunto de tags. Também é possível usar no JavaScript para selecionar uma tag específica.

Onde class é o nome do atributo e intro é o valor daquele atributo.

id=”NomeDoId”

Identificar de forma única um elemento naquela página HTML... É utilizado para identificar destino de âncoras, labels e outras funcionalidades neste sentido.

src=”Link ou diretório da mídia"

Comumente utilizado para indicar para a tag qual arquivo ou mídia utilizar. Recebe valores como links (https://google.com/minhaimagem.jpeg) ou o nome de um arquivo já presente no projeto (/minhaimagem.jpeg).

alt=”Texto alternativo”

O atributo alt fornece informações alternativas para uma imagem se um usuário por algum motivo não puder visualizá-la (devido à conexão lenta, um erro no atributo src ou se o usuário usar um leitor de tela).

href=”Url”

Para a tag <"a">, o atributo href especifica a URL da página para a qual o link vai.

lang=”Linguagem”

O atributo lang especifica o idioma do conteúdo da tag.

Os exemplos comuns são "en" para inglês, "es" para espanhol, "fr" para francês e assim por diante.

target=”blank”

Esse atributo abre o link do documento em uma nova janela ou aba.

Executando uma página HTML

(Voltar)

A função da linguagem HTML (linguagem de marcação de hipertexto) é estruturar o conteúdo de uma página de modo que ela possa ser interpretada e exibida por meio de um navegador web. Os arquivos que contêm linhas escritas em HTML, geralmente, possuem a extensão “.html”. Qualquer navegador web, mesmo em dispositivos móveis, é capaz de abrir um arquivo HTML.

Parte 1 - Salvando um arquivo HTML

  1. Entenda o HTML.strong A sigla significa “hypertext markup language” (linguagem de marcação de hipertexto). As páginas HTML são arquivos de texto que representam tanto o conteúdo quanto o layout do que é visto na tela do navegador web ao se acessar um site ou abrir um arquivo “.html” no computador. Para ver o código contido no arquivo, basta abri-lo em um editor de texto simples, como o Bloco de Notas do Windows. Contudo, um editor profissional também pode ser usado (o Sublime, por exemplo). Por outro lado, se quiser ver a página e não o código, abra o mesmo arquivo em um navegador web (ele será exibida como se estivesse na internet).
  2. Digite ou copie um trecho de código para dentro do Bloco de Notas. Com o tempo, teste outros editores, específicos para programar em HTML, como o Adobe Dreamweaver, o Microsoft Expression Web e o Coffee Cup HTML Editor. Por enquanto, use o Bloco de Notas do Windows ou o TextEdit do Mac.
  3. Salve o arquivo como “nome_do_arquivo.html”. Não se esqueça de sempre salvar o arquivo com a extensão correta. No Bloco de Notas, antes de salvar, mude o “tipo de arquivo” para “todos os arquivos”.

Parte 2 - Executando um arquivo HTML

  1. Confira se há um navegador web instalado em seu computador. Procure pelo Firefox, Chrome, Edge, Safari, Internet Explorer, etc. Se necessário, baixe um.
  2. Encontre o arquivo que salvou. Caso não se lembre em que pasta o salvou, faça uma busca completa. Digite o nome do arquivo na caixa de buscas que existe na base do menu Iniciar (no Windows) ou na caixa de buscas do aplicativo Finder (no Mac).
  3. Clique com o botão direito (Windows) ou dê um clique duplo (Mac) no arquivo e escolha a opção “Abrir com”. O navegador padrão do seu computador estará no topo da lista de programas que podem executar o arquivo. Escolha qualquer navegador, caso haja mais de um. Por exemplo: Firefox, Chrome, Internet Explorer, Safari, etc
  4. Veja o HTML no navegador. O trabalho do navegador é interpretar o código escrito em texto e transformá-lo em uma página visualmente estruturada. Confirme na barra de endereços do navegador se o arquivo aberto é, realmente, o seu.
  5. Tente um método alternativo. Abra o navegador e depois pressione as teclas CTRL + O simultaneamente no teclado. Com isso, deverá aparecer a caixa de diálogo “Abrir”. Encontre o arquivo, clique nele e depois no botão abrir. Caso o atalho de teclado não funcione, vá até a barra de menus, escolha o menu “Arquivo” e depois a opção “Abrir”. Se o menu estiver invisível, pressione a tecla ALT no teclado.

    6. Parte 3 - Executando um arquivo HTML remotamente

    1. Abra uma ferramenta de FTP. Se possível, utilize a ferramenta disponibilizada pelo seu site de hospedagem. De outro modo, use uma de sua escolha, como o FileZilla, por exemplo
    2. Conecte-se ao servidor onde será hospedado o arquivo. Este Passo não é necessário se você estiver usando o gerenciador de arquivos do site de hospedagem. Do contrário, crie uma conexão com os dados fornecidos pelo site, como IP, nome do usuário, senha e porta de conexão.
    3. Arraste o arquivo de seu computador para a área de FTP. O upload começará imediatamente. Caso esteja usando o gerenciador de arquivos do site, abra a pasta “htdocs” (ou a pasta pública) e arraste o seu arquivo para dentro dela.
    4. Espere a conclusão do upload. Se tudo deu certo, basta acessar a página digitando o domínio fornecido pelo site de hospedagem seguido de uma barra e o nome do seu arquivo. Caso apareça uma lista de arquivos, clique no seu “.html”.

    Textos

    (Voltar)

    Um dos principais objetivos do HTML é dar estrutura de texto e significado, também conhecido como semântica), para que um navegador possa exibi-lo corretamente.

    O básico: Cabeçalhos e Parágrafos

    O texto mais estruturado é composto por títulos e parágrafos,esteja você lendo uma história, um jornal, um livro da faculdade, uma revista, etc.

    O conteúdo estruturado torna a experiência de leitura mais fácil e mais agradável.

    Em HTML, cada parágrafo deve ser envolvido em um elemento <'p'> , assim e cada título deve ser envolvido em um elemento de título

    Existem seis elementos de título — <'h1'> (en-US), <'h2'> (en-US), <'h3'> (en-US), <'h4'> (en-US), <'h5'> (en-US) e <'h6'> (en-US) . Cada elemento representa um nível diferente de conteúdo no documento; <'h1'> representa o título principal, <'h2'> representa subtítulos, <'h3'> representa sub-subtítulos, e assim por diante:

    • Preferencialmente, você deve usar apenas um <'h1'> por página — esse é o título do nível superior e todos os outros ficam abaixo dele, na hierarquia.
    • Certifique-se de usar os títulos na ordem correta na hierarquia. Não use <'h3'> para representar subtítulos, seguido de <'h2'>para representar sub-subtítulos - isso não faz sentido e levará a resultados estranhos.
    • Dos seis níveis de cabeçalho disponíveis, você deve usar no máximo três por página, a menos que considere necessário usar mais. Documentos com vários níveis (ou seja, uma hierarquia profunda de cabeçalho) tornam-se difíceis de navegar. Nessas ocasiões, é aconselhável espalhar o conteúdo por várias páginas, se possível.
